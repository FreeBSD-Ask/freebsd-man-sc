# pbio.4.i386

`pbio` — 8255 并行外设接口基础

## 名称

`pbio` I/O 驱动

## 概要

`device pbio`

`在 /boot/device.hints 中：hint.pbio.0.at="isa" hint.pbio.0.port="0x360"`

`#include <dev/pbio/pbioio.h>`

## 描述

`pbio` 驱动支持对在模式 0（简单 I/O）下运行的 Intel 8255A 可编程外设接口（PPI）芯片的直接访问。此类接口提供 24 条数字 I/O 线。该驱动设计用于在使用 Advantech PCL-724 卡等外设时在程序控制下执行 I/O，该卡模拟模式 0 下的 Intel 8255A PPI。其他基于 8255A 的外设（如 BMC Messsysteme PIO24II 卡）也有报告称可正常工作。

PPI 提供两个 8 位端口（端口 A 和端口 B）以及两个 4 位端口（端口 C 上半部、端口 C 下半部）。每个端口都可以单独编程用于输入和（锁存）输出，并出现在设备基地址的不同偏移处。

一个单独的寄存器允许配置端口用于输入或输出。该设备非常简单，当输入数据到达其端子时无法可靠探测；因此，内核配置必须指定设备的基地址。设备驱动程序提供四个字符设备，对应外设的 I/O 端口。打开设备进行读取或写入会自动将相应的硬件端口配置为输入或输出。在引导时，所有端口都配置为输入，以避免损坏外部电路。

一组 ioctl(2) 请求允许在驱动程序级别高效地执行轮询输入和定速输出，而无需昂贵的用户/内核上下文切换。驱动程序可以三种不同方式执行 I/O：

**Basic** 读或写操作在以总线速度将数据读取或写入端口后立即返回。

**Paced** 数据按单独的 ioctl(2) 调用指定的时间间隔从端口传输或传输到端口。

**Differential** （仅输入。）仅返回与先前端口值不同的端口值。

定速间隔以 *Hz* 为单位指定。设置 `n` 秒的步长将导致每 `n` 秒最多读取或写入一个值。单字节读/写操作至少需要 `n` 秒才能完成。

支持以下 ioctl(2) 调用：

**`PBIO_SETDIFF`** 接受一个指向整数的指针作为第三个参数，如果该整数非零，则将驱动程序设置为差分输入。输入步进速度决定驱动程序检查端口以查找已更改值的周期性间隔。

**`PBIO_GETDIFF`** 接受一个指向整数的指针作为第三个参数，并将该整数设置为差分输入的最后设置值。

**`PBIO_SETIPACE`** 接受一个指向整数的指针作为第三个参数，并将驱动程序的输入步进速度设置为该整数的值。

**`PBIO_GETIPACE`** 接受一个指向整数的指针作为第三个参数，并将该整数设置为输入步进的最后设置值。

**`PBIO_SETOPACE`** 接受一个指向整数的指针作为第三个参数，并将驱动程序的输出步进速度设置为该整数的值。

**`PBIO_GETOPACE`** 接受一个指向整数的指针作为第三个参数，并将该整数设置为输出步进的最后设置值。

## 文件

**`/dev/pbio0a`** 端口 A（8 位 I/O）。
**`/dev/pbio0b`** 端口 B（8 位 I/O）。
**`/dev/pbio0ch`** 端口 C 上半部（4 位 I/O）。
**`/dev/pbio0cl`** 端口 C 下半部（4 位 I/O）。

## 缺陷

PCL-724 卡的输入之一可以选择性地接线以产生中断。不支持此功能。
