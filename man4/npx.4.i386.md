# `npx(4)`

`npx` — 数值处理扩展协处理器

## 名称

`npx`

## 概要

`device npx hint.npx.0.at="nexus" hint.npx.0.port="0x0F0" hint.npx.0.flags="0x0" hint.npx.0.irq="13"`

## 描述

`npx` 驱动启用系统数值处理扩展协处理器。数值处理扩展存在于配备 486DX CPU 的系统以及配备 387 或 487SX 协处理器的系统中。`npx` 驱动是系统正常运行所必需的。如果系统中没有 NPX，系统将无法引导。

`npx0` 的标志为：

**0x01** 不要使用 NPX 寄存器来优化 bcopy。

**0x02** 不要使用 NPX 寄存器来优化 bzero。

**0x04** 不要使用 NPX 寄存器来优化 copyin 或 copyout。

当满足以下所有条件时，通常会使用 NPX 寄存器优化复制和清零操作：

- `cpu I586_CPU` 是一个选项
- CPU 是 i586（可能不是 Pentium）
- 对 `npx0` 的探测成功
- INT 16 异常处理工作正常

此时使用 NPX 寄存器进行复制和清零通常快 30-100%。

可以使用这些标志来控制不工作或更慢的情况。在引导时使用 hints 设置这些标志可以正确工作（优化在引导过程后期 `npx0` 附接之后才会使用）。

## 缺陷

缺陷很多，尤其是在老旧廉价的主板上。特别是有些主板上 NPX 到 CPU 的中断线路连接不正确。
